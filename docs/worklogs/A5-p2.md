## 关键决策

1. **Client ≠ retry loop**. Per ADR-001 §4.3 all retry/backoff lives at the
   Prefect layer via `tags=["fish-api"]` + `retries=3`. The client classifies
   HTTP outcomes into typed exceptions (`FishAuthError` / `FishRateLimitError`
   / `FishServerError` / `FishClientError`) and lets Prefect's retry policy
   decide. Network-level `httpx.TimeoutException` / `TransportError` are
   intentionally allowed to bubble up unchanged.

2. **Module-level DI instead of threading dependencies through the task
   signature**. Prefect tasks serialize their parameters into the flow run
   record, so passing a live `AsyncSession`/`MinIOStorage` would be wrong.
   The worker startup hook (owned by A8-Flow) is expected to call
   `configure_p2_dependencies(...)` once per process; tests do the same in
   an `autouse=True` fixture that also tears state down.

3. **Pure coroutine split**. `run_p2_synth(chunk_id, params)` contains the
   actual logic and is directly `await`-able from tests. `p2_synth` is a
   3-line Prefect-decorated wrapper. This avoids needing a Prefect runtime
   in the unit suite, which would blow up the test dependency surface.

4. **Take id = ULID (lexicographically sortable)**. Falls back to `uuid4`
   if the `ulid` package is missing. Chose ULID because it keeps future
   `ORDER BY id` queries on `takes` cheap, matching ADR-001 §5.1 which
   already types `takes.id` as `TEXT`.

5. **WAV duration via stdlib `wave`**. No new deps. Returns `0.0` on a
   non-parseable payload rather than aborting a successful synth — duration
   is metadata, not a correctness gate.

6. **`stage_failed` is best-effort**. If the event write itself fails, we
   log and re-raise the *original* exception so Prefect still sees the real
   failure class. Event observability must never mask task failures.

7. **Params precedence**: explicit dict > env vars > `FishTTSParams`
   defaults. Implemented in `build_params_from_env` which is the only place
   coupled to `os.environ`. Keeps `FishTTSParams` itself environment-free
   so it is safe to serialize into `takes.params`.

8. **`autouse` fixture teardown** resets `_session_factory` / `_storage` /
   `_fish_client_factory` to `None` between tests so that test isolation
   survives module-level DI state.

## 放弃的方案

- **Token bucket / semaphore inside `FishTTSClient`**. Discarded explicitly;
  ADR-001 §4.3 already locks in Prefect concurrency limits as the single
  throttle authority.
- **Sharing `httpx.AsyncClient` as a module singleton**. Rejected because
  it leaks between tests; the client now optionally creates its own client
  lazily but tests always pass in a transport-backed one.
- **Rewriting P2 logic cross-referenced against `scripts/p2-synth.js`**.
  Forbidden per W0 greenfield rule. Did not read it.
- **Writing a `stage_run` row**. Not required by the task brief — the
  `stage_runs` table is documented in ADR-001 §5.1 but the P2 contract
  only asks for take insert + chunk status + events. Left as an A8-Flow
  concern where attempt counting / finish time belong.

## 给下游的提示 (A8-Flow, A9-API)

1. **Concurrency limit must be registered before first run**:
   `prefect concurrency-limit create fish-api 3` (number per Fish plan).
   A8 should own this as part of `server/flows/concurrency.py` per ADR-002
   §4.5. The task tag `"fish-api"` is the exact string to match.

2. **Dependency wiring**: Prefect worker startup (the script A8 will land
   in `server/flows/deploy.py`) must call
   `server.flows.tasks.p2_synth.configure_p2_dependencies(
       session_factory=..., storage=..., fish_client_factory=...
   )`. Without this the task raises `RuntimeError`.

3. **FishAuthError is fatal, but Prefect still retries**. By default
   `retries=3` will retry *any* exception. If you want to short-circuit on
   auth errors, wrap the task call inside a flow with a state handler or
   use Prefect 3's `retry_condition_fn` to skip retries for `FishAuthError`.
   Decided not to hardcode this in the task itself because the policy is
   arguably a flow-level concern.

4. **Per-chunk interface**: `p2_synth(chunk_id: str, params: dict | None)`.
   The flow's `.map()` over P1 results should extract `chunk.id` and pass
   it positionally.

5. **`takes.params` column** now contains the full `FishTTSParams.model_dump()`
   payload (temperature / top_p / model / reference_id / etc). This is the
   contract A9-API will expose in its take detail route.

### Mock-covered error scenarios

| Scenario | Exception | Retry semantics |
|---|---|---|
| HTTP 200 + WAV body | — | success |
| HTTP 200 + empty body | `FishClientError` | fatal |
| HTTP 401 | `FishAuthError` | fatal per ADR §4.3 |
| HTTP 403 | `FishAuthError` | fatal |
| HTTP 429 | `FishRateLimitError` | Prefect retry |
| HTTP 500 | `FishServerError` | Prefect retry |
| HTTP 400 | `FishClientError` | fatal |
| `httpx.TimeoutException` | raw exception | Prefect retry |
| Empty input text | `FishClientError` (no HTTP call) | fatal |
| Empty api_key at construction | `ValueError` | fatal |
| Chunk missing (DB) | `DomainError("not_found")` | fatal |
| Empty `text_normalized` | `DomainError("invalid_input")` | fatal |
| MinIO upload failure | upstream exception passthrough | Prefect retry |
