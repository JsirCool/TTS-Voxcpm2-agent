## 关键决策

### 1. `build_ffmpeg_concat_list` 签名收窄

A7 brief 描述的是 `(audio_files, silences: dict[float, Path])` — 但这让纯函数
多承担了"插值"的职责且无法在不知道 gap 序列的前提下组合出正确顺序。拆成两个
独立的纯函数更干净:

- `interleave_with_silences(audio_files, gaps, silences)` — 负责排列顺序。
- `build_ffmpeg_concat_list(entries)` — 只做路径转义 + 序列化。

单元测试仍然覆盖了 brief 指定的 4 条验收项(路径转义 / 顺序保留 / 零 gap
跳过 / gap 键缺失抛错),只是分布在两个函数上。这是有意偏离。

### 2. `run_p6_concat` 与 Prefect wrapper 分离

Prefect `@task` 自身不接受注入,但测试必须能跑(W3 brief 要求"单测必须能跑"
且"不假设上下文有 prefect runtime")。因此把业务全部写进
`run_p6_concat(session, storage, ...)`,Prefect 装饰函数只负责 engine 和
MinIO client 生命周期。集成测直接调用 `run_p6_concat`,绕过 Prefect runtime。

### 3. 零时长 chunk 处理

brief 写"跳过,但记录 warning"。把过滤放在 offset 计算之前,否则
offset 序列会引入一个 0 秒块的 gap,得到错误总长。过滤后如果 `timings`
全空,升级为 `DomainError("invalid_state")`。

### 4. `-c copy` 的前置条件

brief 写"全部 chunk 假设是 16kHz/mono/16bit WAV"。为了保证静音段也吻合
这一假设,`generate_silence` 固定用 `pcm_s16le` 编码且默认采样率 16k /
单声道,与 P2 输出严格一致。未来 P2 改输出格式需要同步这两行。

### 5. SRT 合并时间偏移算法

`merge_srt_files(srt_strings, offsets)` 是纯字符串函数:

1. 每个 SRT 字符串用 `parse_srt` 解析成 `SrtCue` 列表。
2. 所有 cue 按 chunk 顺序 flatten,每个 cue 的 start/end 加上对应 chunk 的
   `offset`。
3. 从 1 开始重新编号,统一用 `format_srt_timestamp` 输出
   `HH:MM:SS,mmm`(逗号分隔,最近毫秒四舍五入,负值钳到 0)。

offset 计算是 *累加* 模型:`offsets[i] = offsets[i-1] + dur[i-1] + gap[i-1]`,
其中 `gap` 在 same-shot 用 `padding_s`,cross-shot 用 `shot_gap_s`。第一个
chunk 永远从 0 开始,不插前导静音。

### 6. 可重入

MinIO `put_object` 是 last-write-wins,`upload_file(final_wav_key(...))`
天然覆盖。DB 端重跑只是再写一条 `stage_started`/`stage_finished` event +
再把 `episodes.status` set 成 `done`,均为幂等。工作目录用 `tempfile.mkdtemp`
并在 finally 里清理。

## 关键决策记录:chunk_offset / SRT 偏移算法

```
offsets[0] = 0
for i in 1..n:
    gap_i = shot_gap_s if chunks[i].shot_id != chunks[i-1].shot_id else padding_s
    offsets[i] = offsets[i-1] + chunks[i-1].duration_s + gap_i

total = offsets[-1] + chunks[-1].duration_s
```

对每个 chunk 的 SRT:`new_cue.start = old_cue.start + offsets[i]`,结束
时间同理。全 episode 的 cue 从 1 重新编号,格式 `HH:MM:SS,mmm`。

不引入前导/尾随静音(brief 没要求),第一个 chunk 从 0s 开始,最后一个
chunk 的结尾就是 episode 总长。
