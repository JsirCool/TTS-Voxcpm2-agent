## 关键决策

### 1. 字符加权 > 词级对齐

WhisperX 给的是 **音频词**的时间戳，而字幕行是**显示文本**的句子——两者不
是 1:1 的：控制标记被 strip 掉、phoneme 标记替换发音、中英混排的 token 切
分口径也不一样。试图按"word 精对齐到字幕字符"反而会在边界上抖动。

所以 P5 采用的是"**信任 take 的总时长，按字幕行的字符数加权**"的简单策略：

```
d_i = len(line_i) / sum(len(line_j)) * total_duration
```

优点：
- 确定性到 bit 级；同输入永远出同一个 SRT
- 不依赖 WhisperX word 数组的具体形状，只把它当"非空"校验的 sentinel
  （empty transcript → `invalid_state`）
- 单 take 连续朗读的场景下视觉效果足够好（字幕切换节奏 ≈ 朗读节奏）

代价：如果朗读某句特别快/特别慢（韵律 outlier），字幕会微微错位。需要更
精细对齐时，后续可以用 WhisperX word 做二次 refinement（在同一个 pure 函数
里追加参数，不破坏现有契约）。

### 2. 用 take.duration_s，不用 transcript.duration_s

`WhisperXTranscript` 的 `duration_s` 只是信息字段。真正的"音频有多长"以
`takes.duration_s`（P2 写入时用 `wave` 模块解析 WAV header）为准。如果
WhisperX 的 duration 和 take 不一致，以 take 为准能保证字幕不超过实际音频
长度（SRT 的最后一句被 snap 到 `total_duration` 整数值，进一步消除 float
drift）。

### 3. `p5_done` 作为新的 chunk 状态值

产品需求说 "chunk.status → p5_done"。`domain.ChunkStatus` Literal 里没
这个 token，但 `ChunkRepo.set_status` 实际写的是纯 `Text` 列，Literal 只
影响静态类型检查不影响运行时。我没有扩 Literal 避免和 A2/A9 的契约冲突；
如果 A9-API 要在 serializer 里暴露状态字段，届时统一补齐。**给 A9 的提
示**：如果 API 要返回 ChunkStatus，要把 `p5_done` 加到 Literal。

### 4. `DomainError` 升到 `server.core.domain`

A5 在 `p2_synth.py` 里本地定义了 `DomainError`（那是 A2-Domain 没提供
时的权宜之计）。我把它搬到 `domain.py` 作为真正的共享类，带 `code` /
`message` 双属性；P5 task 和 logic 都用这个类。P2 的局部定义还保留（我不
能改 p2_synth.py），两个类实例不是同一个，但 A8-Flow 在编排时如果要按 code
分支可以用 `isinstance(exc, DomainError)` + `getattr(exc, "code", None)`。
**给 A8/A9 的提示**：新代码统一用 `server.core.domain.DomainError`，必要
时把 p2_synth 的本地类换成 re-export。

### 5. `compose_srt` 作为 "pure orchestrator"

把 strip / split / distribute / build 串起来的编排写在 `p5_logic.py` 里而
不是 task 里。好处是 task 缩成 "read transcript → call compose_srt →
upload SRT" 的薄壳，所有算法决策都在 pure 层，单测覆盖成本极低（同一个
`compose_srt` 调用在 logic 测和 task 测两边都能复用断言）。

## 放弃的方案

- **用 `srt` 第三方库**：加依赖 vs 20 行自己写的 `_format_ts` + `build_srt`，
  自己写更可控。ADR-002 §4.1 也说依赖清单只能通过 ADR 变更流程扩充。
- **strip 阶段按白名单 `[break|long break|breath|...]` 匹配**：改成通配
  `\[[^\[\]]*\]` 贪婪 strip。原因：authoring convention 规定显示文本里不
  会有字面方括号，所以任何 `[...]` 都可以扔；白名单会漏掉 `[sigh]`
  `[^pronounce]` 这种延后加入的 marker。
- **按逗号/分号也切分字幕行**：会产生太多碎片，不好读；只在句终止符处切。
- **用 word-level 时间戳做"真对齐"**：见决策 1，放弃。

## 给下游的提示 (A8-Flow)

1. **Task 签名**：`run_p5_subtitles(chunk_id: str) -> P5Result`。Prefect
   包装器名字是 `p5-subtitles`，`retries=2`, `retry_delay_seconds=[2,8]`。
2. **DI**：worker 启动时调用 `configure_p5_dependencies(session_factory=,
   storage=)`。用 `async_sessionmaker` 作为 `session_factory`，P5 会自己
   `async with factory()` 获取 session。
3. **状态前置**：P5 假设 `chunk.status == "transcribed"` 且
   `chunk.selected_take_id` 非空、transcript 在 MinIO。P3 task 要保证
   这几个条件满足后再让 P5 排队。
4. **新状态值**：P5 成功后 `chunk.status = "p5_done"`。A8 编排时 P6 的
   guard 可以用这个。
5. **事件语义**：成功写 `stage_started` + `stage_finished` 两条，失败写
   `stage_started` + `stage_failed`。SSE 客户端按 `payload.stage == "p5"`
   过滤即可。
6. **确定性**：同 chunk_id + 同 take + 同 transcript → 同一份 SRT 字节。
   可以放心在 retry-chunk mini flow 里裸 re-run。
